Introduction

Music is more than just sound; it is data. In this project, we dive into the world of music analytics to understand what makes a song “popular,” “danceable,” or fit for a specific mood. By analyzing a dataset of song characteristics and a collection of user-generated playlists, we aim to uncover trends in listening habits.

Our final goal is to leverage these insights to curate the Ultimate Playlist—a data-driven selection of tracks designed to maximize listener engagement through sonic continuity and energy flow.

Data Acquisition and Cleaning

To conduct this analysis, we must first ingest data from two distinct sources: a library of song metrics and a set of user playlists. To ensure reproducibility and responsible resource usage, we implement functions that strictly download data only when it is not already present locally.

Task 1: Song Characteristics Dataset

This dataset provides technical metrics (danceability, tempo, key, etc.) for a vast library of songs. We clean the artists column to ensure every artist is individually analyzable.

Code 
load_songs <- function() {
  data_dir <- "data/mp03"
  file_path <- file.path(data_dir, "songs.csv")
  url <- "https://raw.githubusercontent.com/gabminamedez/spotify-data/master/data.csv"
  
  if (!dir.exists(data_dir)) dir.create(data_dir, recursive = TRUE)
  
  if (!file.exists(file_path)) {
    message("Downloading song data...")
    download.file(url, destfile = file_path, mode = "wb")
  }
  
  songs_raw <- read_csv(file_path, show_col_types = FALSE)
  
  clean_artist_string <- function(x) {
    x |>
      str_replace_all("\\['", "") |>
      str_replace_all("'\\]", "") |>
      str_replace_all("'", "") |> 
      str_replace_all(", ", ",")
  }
  
  songs_clean <- songs_raw |>
    mutate(artists_clean = clean_artist_string(artists)) |>
    separate_longer_delim(artists_clean, delim = ",") |>
    mutate(artist = artists_clean) |>
    select(-artists, -artists_clean)
  
  return(songs_clean)
}

Task 2: Playlist Dataset

The Spotify Million Playlist dataset is massive. For this mini-project, we download a subset (slices of 1,000 playlists) and process them into a usable format.

Code 
load_playlists <- function(n_slices = 1) {
  data_dir <- "data/mp03"
  if (!dir.exists(data_dir)) dir.create(data_dir, recursive = TRUE)
  
  base_url <- "https://raw.githubusercontent.com/Devinogrady/spotify_million_playlist_dataset/master/data1/"
  
  all_playlists <- list()
  
  for (i in 0:(n_slices - 1)) {
    start_idx <- i * 1000
    end_idx <- start_idx + 999
    filename <- paste0("mpd.slice.", start_idx, "-", end_idx, ".json")
    file_path <- file.path(data_dir, filename)
    file_url <- paste0(base_url, filename)
    
    # Responsible Download
    if (!file.exists(file_path)) {
      message(paste("Downloading", filename, "..."))
      tryCatch({
         download.file(file_url, destfile = file_path, mode = "wb")
      }, error = function(e) {
        warning(paste("Failed to download", filename, ":", e$message))
      })
    }
    
    # Read and append to list
    if (file.exists(file_path)) {
      json_data <- fromJSON(file_path, simplifyVector = TRUE)
      if ("playlists" %in% names(json_data)) {
        all_playlists[[length(all_playlists) + 1]] <- as_tibble(json_data$playlists)
      }
    }
  }
  
  return(all_playlists)
}

raw_playlists_list <- load_playlists(n_slices = 2)

Task 3: Rectangling the Data

The playlist data arrives in a nested JSON structure. We “rectangle” it into a tidy dataframe. Per the project requirements, we also ensure we capture the Playlist Position and handle potential missing columns.

Code 
strip_spotify_prefix <- function(x) str_remove(x, "spotify:[a-z]+:")

playlists_df <- map_dfr(raw_playlists_list, function(slice) {
  if (nrow(slice) > 0 && "tracks" %in% names(slice)) {
    # Unnest with unique names to prevent collision
    expanded_slice <- slice |> 
      unnest(tracks, names_repair = "unique") |>
      group_by(pid) |>
      mutate(playlist_position = row_number()) |>
      ungroup()
    
    # Find the duration column dynamically
    dur_col <- names(expanded_slice)[str_detect(names(expanded_slice), "^duration_ms")]
    
    if (length(dur_col) > 0) {
       expanded_slice <- expanded_slice |> rename(duration_ms_std = !!sym(dur_col[1]))
    } else {
       expanded_slice$duration_ms_std <- NA_real_
    }
    
    # Ensure all required columns exist
    required_cols <- c("track_uri", "artist_uri", "album_uri", "track_name", 
                       "artist_name", "album_name")
    for (col in required_cols) {
      if (!col %in% names(expanded_slice)) expanded_slice[[col]] <- NA
    }

    expanded_slice |>
      select(
        playlist_name = name,
        playlist_id = pid,
        playlist_position,
        playlist_followers = num_followers,
        track_name,
        track_id = track_uri,
        artist_name,
        artist_id = artist_uri,
        album_name,
        album_id = album_uri,
        duration_ms = duration_ms_std
      )
  } else {
    tibble()
  }
}) |>
  mutate(
    track_id = strip_spotify_prefix(track_id),
    artist_id = strip_spotify_prefix(artist_id),
    album_id = strip_spotify_prefix(album_id),
    duration_ms = as.numeric(duration_ms)
  )

Building the Ultimate Playlist

Task 6: Finding Related Songs

To build our playlist, we select “Midnight City” by M83 as our anchor. This track epitomizes the “driving at night” aesthetic we aim to curate. We use heuristics including Key matching, Tempo range (± 15 BPM), and Vibe matching (Valence/Energy) to find suitable candidates.

Code 
anchor_song_name <- "Midnight City"
anchor <- combined_data |> 
  filter(name == anchor_song_name) |> 
  slice(1)

# Fallback if anchor is missing
if (nrow(anchor) == 0) {
  anchor <- combined_data |> 
    arrange(desc(popularity)) |> 
    slice(1)
}

find_candidates <- function(data, anchor_song) {
  data |>
    filter(
      name != anchor_song$name,
      key == anchor_song$key,
      between(tempo, anchor_song$tempo - 15, anchor_song$tempo + 15),
      between(valence, anchor_song$valence - 0.2, anchor_song$valence + 0.2)
    )
}

candidates <- find_candidates(combined_data, anchor) |>
  mutate(is_popular = popularity > 50)

Task 7: Curate and Analyze

We select a final list of 12 songs: the anchor plus a mix of popular hits and “hidden gems” (low popularity).

Code 
set.seed(123)  # For reproducibility

final_playlist <- bind_rows(
  anchor,
  candidates |> filter(!is_popular) |> slice_sample(n = min(6, sum(!candidates$is_popular))), 
  candidates |> filter(is_popular) |> slice_sample(n = min(5, sum(candidates$is_popular))) 
) |> 
  distinct(name, .keep_all = TRUE)

Deliverable: The Ultimate Playlist

Playlist Title: “Neon Midnight Drive”

Description: A synth-heavy, driving playlist designed for late-night coding sessions or highway drives. It blends high-energy electronic beats with melancholic undertones, ensuring the listener stays awake but not overwhelmed.

Design Principles:

  1. Sonic Continuity: All tracks share the same musical key (11) and tight tempo range to allow for seamless cross-fading.
  2. The “All Rise” Effect: As visualized in the flow analysis, the energy maintains a steady upward trend to prevent fatigue.
  3. Discovery: Over 50% of the tracks are “Hidden Gems” (Popularity < 50), offering listeners new sounds alongside familiar hits.
